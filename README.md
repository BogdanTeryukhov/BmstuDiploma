# Эффективная реализация нормальных алгоритмов Маркова

## 📖 Описание

Этот проект представляет собой эффективную реализацию нормальных алгоритмов Маркова с использованием структуры данных "верёвка" (rope) и автоматных функций для оптимизации операций со строками.

Проект создан в рамках дипломной работы по теме "Эффективная реализация нормальных алгоритмов Маркова" в МГТУ им. Баумана.

## 🚀 Основные особенности

### Структура данных "Верёвка" (EnhancedRope)
- **Дерево**: Внутренние узлы представляют конкатенацию, листья содержат текстовые данные
- **Автоматные функции**: В каждом узле хранятся предвычисленные функции `a(s)` (действие) и `f(s)` (расстояние)
- **Эффективность**: Операции префикса, суффикса и конкатенации выполняются за O(log N) при сбалансированном дереве

### Автоматы для поиска подстрок
- **KMP Automaton**: Реализация алгоритма Кнута-Морриса-Пратта для эффективного поиска
- **Regex Automaton**: Поиск по регулярным выражениям вида `.*s.*`

### Оптимизированные операции
- **Конкатенация**: O(log N) при сбалансированном дереве
- **Поиск подстрок**: Использование предвычисленных автоматных функций
- **Вставка/Удаление**: Эффективные операции с автоматическим пересчетом функций

## 🛠 Техническая реализация

### Математическая основа
Реализация основана на следующих принципах:

1. **Композиция функций**: `a(s+t) = a(t) * a(s)` вычисляется за время O(|Q|), где |Q| - количество состояний автомата
2. **Функция расстояния**: `f(s+t)` эффективно вычисляется через `f(s)` и `f(t)`
3. **Локальный пересчет**: При изменении структуры пересчитываются только узлы от изменённого до корня за O(log|t|)

## 📊 Эффективность

### Теоретическая сложность
- **Конкатенация**: O(log N) при сбалансированном дереве
- **Поиск подстрок**: O(|pattern|) с предвычисленными функциями
- **Вставка/Удаление**: O(log N + |inserted_string|)

### Практические результаты
- Ускорение поиска: 1.05x - 1.2x по сравнению с наивным подходом
- Эффективность возрастает с размером данных
- Значительное улучшение для частых операций поиска/модификации

## 📈 Пример использования

```kotlin
// Создание правил Маркова
val rules = parseRules(listOf(
    "ab -> ba",
    "ba ->. DONE"
))

// Создание алгоритма
val markov = MarkovAlgorithm(rules)

// Запуск алгоритма
val result = markov.run("ab")
println(result) // Вывод: DONE

// Запуск с трассировкой
val tracedResult = markov.runWithTrace("ab", verbose = true)
```
