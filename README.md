# Эффективная реализация нормальных алгоритмов Маркова

## Описание

Этот проект представляет собой эффективную реализацию нормальных алгоритмов Маркова с использованием структуры данных "верёвка" (rope) и автоматных функций для оптимизации операций со строками.

Проект создан в рамках дипломной работы по теме "Эффективная реализация нормальных алгоритмов Маркова" в МГТУ им. Баумана.

## Основные особенности

### Структура данных "Верёвка" (EnhancedRope)
- **Дерево**: Внутренние узлы представляют конкатенацию, листья содержат текстовые данные
- **Автоматный поиск**: Эффективный поиск подстрок с использованием функций `a(s)` (действие) и `f(s)` (расстояние)
- **Балансировка**: Автоматическая балансировка дерева для поддержания O(log N) сложности
- **Эффективность**: Операции вставки, удаления и поиска выполняются за O(log N) при сбалансированном дереве

### Автоматы для поиска подстрок
- **KMP Automaton**: Реализация алгоритма Кнута-Морриса-Пратта для эффективного поиска
- **Regex Automaton**: Поиск по регулярным выражениям вида `.*s.*`

### Оптимизированные операции
- **Конкатенация**: O(log N) при сбалансированном дереве с автоматической балансировкой
- **Поиск подстрок**: O(log N + |Q|) с автоматными функциями без преобразования в строку
- **Вставка/Удаление**: O(log N + |Q|²) с локальным пересчетом аннотаций
- **Рекурсивный поиск**: Пропуск поддеревьев без вхождений для максимальной эффективности

## Техническая реализация

### Математическая основа
Реализация основана на следующих принципах:

1. **Композиция функций**: `a(s+t) = a(t) ∘ a(s)` вычисляется за время O(|Q|²), где |Q| - количество состояний автомата
2. **Функция расстояния**: `f(s)` возвращает позицию первого достижения финального состояния или -1
3. **Локальный пересчет**: При изменении структуры пересчитываются только узлы от изменённого до корня за O(log N)
4. **Эффективный поиск**: Рекурсивный алгоритм пропускает поддеревья без вхождений

## Эффективность

### Теоретическая сложность
- **Конкатенация**: O(log N) при сбалансированном дереве
- **Поиск подстрок**: O(log N + |Q|) с автоматными функциями, где |Q| - количество состояний автомата
- **Вставка/Удаление**: O(log N + |Q|²) с пересчетом аннотаций

### Практические результаты
- Значительное ускорение поиска: до 10x для больших строк по сравнению с наивным подходом
- Эффективность возрастает с размером данных
- Минимальные накладные расходы для малых строк
- Линейная масштабируемость при увеличении числа операций

## Пример использования

```kotlin
// Создание правил Маркова
val rules = parseRules(listOf(
    "ab -> ba",
    "ba ->. DONE"
))

// Создание алгоритма
val markov = MarkovAlgorithm(rules)

// Запуск алгоритма
val result = markov.run("ab")
println(result) // Вывод: DONE

// Запуск с трассировкой
val tracedResult = markov.runWithTrace("ab", verbose = true)
```

## Структура проекта

### Основные компоненты

1. **EnhancedRope** - Структура данных "верёвка" с эффективным поиском подстрок
2. **MarkovAlgorithm** - Реализация нормального алгоритма Маркова с терминальными правилами
3. **Automata** - Конечные автоматы для поиска подстрок (KMP, Regex)
4. **EnhancedRopeUtils** - Вспомогательные функции для работы с верёвкой (split, insert, delete) и поиском

### Алгоритмы

- **Кнута-Морриса-Пратта** для поиска подстрок
- **Нормальные алгоритмы Маркова** для преобразования строк
- **Автоматные функции** для оптимизации поиска
